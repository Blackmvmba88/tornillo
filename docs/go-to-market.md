# ScrewVision Go-To-Market Kit

Creator: **Iyari Cancino Gomez**, alias **BlackMamba**.

## Positioning

ScrewVision turns ordinary cameras into lightweight visual inspection stations for industrial parts.

Short pitch:

```text
I build lightweight computer vision systems for inspection, counting, and quality-control workflows.
```

More direct:

```text
I turn common cameras into intelligent visual inspection stations.
```

## What To Show

Use the current repo as proof that this is more than an idea:

- Python/OpenCV runtime
- FastAPI backend
- React/Vite dashboard
- Detector-neutral response contract
- SQLite telemetry
- Python 3.11/3.12 CI
- Automated GitHub releases
- Packaged WebUI release asset

## Two-Minute Demo

1. Open the dashboard.
2. Capture or upload an image of a screw or industrial part.
3. Show the JSON response with detector, confidence, material estimate, wear, bounding box, and quality score.
4. Show `PASS`, `REVIEW`, or `FAIL` in the dashboard.
5. Show dashboard telemetry: events, detections, confidence, inspection results, and inventory signals.

## Offer

Do not sell "AI". Sell the operational outcome:

```text
I can build a system that detects parts, counts production, flags visual defects, and generates visual reports.
```

Good first customers:

- machine shops
- local manufacturers
- industrial hardware stores
- technical schools
- quality-control labs
- small assembly lines
- inventory-heavy workshops
- auto-parts and CNC businesses

## Pricing Starting Points

These are discovery anchors, not fixed promises:

| Offer | Scope | Price |
| --- | --- | --- |
| Technical demo | One object class, upload-based demo, basic report | MXN 1,500-5,000 |
| Custom prototype | Client-specific object, dashboard, telemetry, offline handoff | MXN 8,000-25,000 |
| Basic installed system | Camera, lighting assumptions, local runtime, reporting | MXN 25,000-80,000+ |

Price drivers:

- object variability
- lighting control
- camera quality
- required accuracy
- number of part classes
- live camera vs upload workflow
- reporting/export requirements
- installation and support
